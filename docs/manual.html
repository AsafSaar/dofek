1. What is dofek?

dofek (Hebrew: דּוֹפֶק — "pulse" or "heartbeat") is a cross-platform system monitor built for developers running local AI workloads. It tracks per-process VRAM, flags Ollama and other inference workloads automatically, and renders CPU history as candlestick charts so you see real variance — not smoothed averages. Supported platforms: Windows 10/11, Linux x86_64, and macOS 12+ on Apple Silicon (aarch64).

It ships as two interfaces driven by the same Rust core:

  • dofek-tui — terminal UI (Ratatui). Needs a truecolor (24-bit RGB) terminal: Windows Terminal, Tabby, GNOME Terminal, Konsole, iTerm2, WezTerm, Ghostty, Alacritty, Kitty. Apple's Terminal.app does not support truecolor — panel backgrounds will render as miscolored blocks. dofek-tui detects non-truecolor terminals at startup and prints a warning before continuing.
  • dofek-gui — desktop app (Tauri 2). WebView2 on Windows, WebKitGTK on Linux, WKWebView on macOS. Same data, same plugins, graphical charts.

2. Quickstart

GUI (desktop app)

  1. Launch dofek:
    • Windows: Start Menu, or dofek.exe from C:\Program Files\dofek\.
    • Linux: application launcher, or run the .AppImage, or dofek-gui after installing the .deb/.rpm.
    • macOS: Launchpad / Applications, or open /Applications/dofek.app. First launch shows "dofek.app is damaged and can't be opened" — the app isn't damaged, it's unsigned and quarantined. Run xattr -dr com.apple.quarantine /Applications/dofek.app once and it launches normally. (On macOS 14 Sonoma and earlier, right-click → Open → Open also works; macOS 15 Sequoia removed that bypass.)
  2. The dashboard opens with live CPU, GPU, memory, and network charts. The process watchlist on the right shows the top VRAM/CPU/MEM consumers.
  3. Click AI / DEV / WATCH tabs to filter the watchlist.
  4. Press ? for the keybinding overlay, or click the Manual link inside the help overlay to reopen this document.

TUI (terminal)

  1. Open your terminal of choice, then run: 
    dofek-tui
  2. With a custom config: 
    # Windows (PowerShell)
dofek-tui --config "$env:APPDATA\dofek\dofek.toml"

# Linux
dofek-tui --config ~/.config/dofek/dofek.toml

# macOS
dofek-tui --config ~/Library/Application\ Support/dofek/dofek.toml
  3. Press ? for help, q to quit. See the full keybinding table.
Font size tip: the TUI is designed for 9–11pt monospace. JetBrains Mono, Fira Code, MesloLGS NF, and Cascadia Code all render the half-block charts correctly. On Windows, run install-wt-profile.ps1 (ships with the source) to add a pre-configured "dofek" profile to Windows Terminal. On Linux/macOS, set the font in your terminal's preferences.

3. TUI vs. GUI

Both interfaces read the same config, run the same plugins, and collect the same data. Pick based on context:

Use caseRecommended
Remote session / SSH / low-resourceTUI
Always-visible dashboard on your desktopGUI
Scripting + snapshot exports (s key)TUI
Smoothed Bezier charts, click-to-killGUI
Running alongside ollama, lm-studioEither — both detect AI workloads

5. Configuration

dofek loads config from the first file it finds, in this order:

  1. --config <path> command-line flag (TUI only)
  2. ./dofek.toml in the current working directory
  3. User config directory — recommended location:
    • Windows: %APPDATA%\dofek\dofek.toml
    • Linux: ~/.config/dofek/dofek.toml
    • macOS: ~/Library/Application Support/dofek/dofek.toml

All sections are optional. Missing values fall back to sensible defaults.

Example config

[general]
refresh_ms = 500          # Poll interval (milliseconds)
history_len = 60          # Chart samples to keep in memory

[display]
show_temps = true
show_power = true
process_count = 50        # Max processes in watchlist

[ai]
vram_threshold_gb = 1.0   # VRAM above this flags a process as AI
known_ai_processes = ["ollama", "ollama_llama_server", "python", "lm_studio", "claude"]

[categories]
dev_processes   = ["code", "cargo", "rustc", "node", "npm", "git", "docker", "go"]
watch_processes = ["ffmpeg", "obs"]
watch_pids      = []

[lhm]
url = "http://localhost:8085"   # LibreHardwareMonitor web server (fallback)

[telemetry]
enabled = false           # Opt-in; disabled by default

[[plugins]]
name       = "ollama"
command    = "dofek-ollama"
args       = ["--host", "http://localhost:11434"]
enabled    = true
timeout_ms = 2000

Option reference

KeyDescription
general.refresh_msPoll interval. Lower = smoother, more CPU. Default 500.
general.history_lenSamples retained per chart. Default 60.
display.show_tempsShow temperature bars in the bottom strip.
display.show_powerShow GPU power draw bars.
display.process_countMax watchlist length; panel height auto-fits.
ai.vram_threshold_gbVRAM above this (per-process) flags the process as AI workload.
ai.known_ai_processesSubstring match (case-insensitive). Matches are treated as AI workloads regardless of VRAM.
categories.dev_processesSubstrings matched against process names for the DEV category.
categories.watch_processesSubstrings pinned to the WATCH category.
categories.watch_pidsSpecific PIDs pinned to WATCH (useful for jobs you're babysitting).
lhm.urlLibreHardwareMonitor web-server URL. Only used if NVML is unavailable.
telemetry.enabledAnonymous usage telemetry. Disabled by default.
[[plugins]]Repeatable table. Each entry spawns a plugin child process.

6. Process categories (AI / DEV / WATCH)

Every process gets classified into zero or one category. Priority goes WATCH → AI → DEV → none.

CategoryHow it's assigned
AI ●Name matches ai.known_ai_processes, or VRAM > ai.vram_threshold_gb, or name ends with _server and uses any VRAM.
DEV ■Name matches categories.dev_processes.
WATCH ★Name matches categories.watch_processes, or PID listed in categories.watch_pids.

Filter the watchlist with keys 1 (ALL) / 2 (AI) / 3 (DEV) / 4 (WATCH), or click the tabs in the GUI.

7. AI workload detection

Once a process is classified as AI, dofek tags its current state:

BadgeCondition
● inferringVRAM > threshold and GPU utilization > 20%.
● loadingVRAM increasing by > 200 MB between polls.
○ idleKnown AI process, low utilization.

The AI badge in the top ticker bar always reflects the currently-most-active AI workload.

8. GPU support & LibreHardwareMonitor

dofek talks to two data sources for GPU metrics:

  1. NVML (primary, all platforms) — NVIDIA driver API. Provides utilization, VRAM, temperature, power, and per-process VRAM. No extra install required beyond the NVIDIA driver.
  2. LibreHardwareMonitor HTTP (Windows-only fallback) — for non-NVIDIA GPUs or when NVML isn't available on Windows.
Per-platform reality: LHM is Windows-only — Linux and macOS users can skip the LHM section below entirely. On Linux without an NVIDIA GPU, dofek shows GPU info as N/A (AMD GPU support is on the roadmap). On macOS Apple Silicon, the GPU is integrated and shares system memory; the GPU tile shows your chip name and points you at the MEM panel — Apple Silicon GPU/VRAM and SMC sensor integration are tracked for a future release.

Setting up LibreHardwareMonitor (Windows only)

  1. Download LibreHardwareMonitor and extract.
  2. Run LibreHardwareMonitor.exe as administrator.
  3. Enable the web server: Options → Remote Web Server → Run (default port 8085).
  4. If you changed the port, update lhm.url in dofek.toml.
Multi-GPU: NVML is fully multi-GPU aware. LHM support is basic — primary GPU only. If you have multiple GPUs and NVML isn't working, open an issue.

9. Plugins

Plugin API is experimental. The JSON protocol is versioned (schema_version: 1) but subject to breaking changes until further notice. Pin your plugin to a specific dofek version if stability matters.

Plugins are external processes that dofek spawns and polls via newline-delimited JSON over stdin/stdout. Each poll cycle, dofek writes a request, reads a response, and renders any panels / process annotations / metrics the plugin returned.

Built-in plugins

PluginPurposeRequires
dofek-ollamaShows loaded models + inference state, annotates Ollama processesOllama running locally
dofek-dockerLists running containers, annotates Docker processesDocker Desktop with TCP API enabled

Enable a plugin by adding a [[plugins]] table to dofek.toml. See the example config above.

Plugin health dock

The plugin dock (inside the watchlist panel) shows one row per plugin:

  • ● healthy
  • ● unhealthy (5+ consecutive errors)
  • ● crashed — dofek restarts with exponential backoff (1s → 30s cap)
  • ○ starting

Plugin development docs: dofek.dev/plugins.

10. Telemetry

dofek includes opt-in anonymous telemetry to help improve the app. It is disabled by default. Nothing is sent unless you explicitly enable it.

What's collected (if you opt in)

  • Session duration and interface (TUI vs. GUI)
  • Feature usage: which tabs, filters, modes you use
  • GPU detection path (NVML / LHM / none) and device count
  • Periodic heartbeats (process count, active tab)

What's never collected

  • Process names, hostnames, IP addresses, or any identifying information
  • System metric values (CPU %, memory usage, etc.)
  • Config file contents or custom process lists

A random anonymous UUID is generated on first run and stored alongside dofek.toml in the user config directory: %APPDATA%\dofek\settings.toml on Windows, ~/.config/dofek/settings.toml on Linux, ~/Library/Application Support/dofek/settings.toml on macOS. Events batch locally and flush via HTTPS every 60 seconds. If the endpoint is unreachable, batches are silently dropped.

To disable: toggle the switch in the GUI help overlay, or set telemetry.enabled = false in dofek.toml.

11. Troubleshooting

OS warning on first run (unsigned binary)

Binaries are currently unsigned. Code signing is on the roadmap. Workarounds per OS:

  • Windows (SmartScreen): right-click the installer or executable → Properties → check Unblock at the bottom of the General tab → OK.
  • Linux (AppImage): chmod +x dofek_*.AppImage before running. .deb/.rpm packages install normally via apt/dnf.
  • macOS (Gatekeeper): first launch shows "dofek.app is damaged and can't be opened". The app isn't damaged — it's unsigned and quarantined by your browser. Run xattr -dr com.apple.quarantine /Applications/dofek.app once. On macOS 14 Sonoma and earlier you can also right-click → Open → Open; macOS 15 Sequoia removed that bypass.

Verify the download first by comparing against SHA256SUMS.txt from the GitHub release:

# Windows (PowerShell)
Get-FileHash .\dofek_1.3.4_x64_en-US.msi -Algorithm SHA256

# Linux
sha256sum -c SHA256SUMS.txt

# macOS
shasum -a 256 -c SHA256SUMS.txt

GPU panel shows "no devices" (or your chip name on Apple Silicon)

  • NVIDIA GPU? Make sure drivers are current. nvidia-smi should work from a terminal.
  • AMD / Intel GPU? NVML won't work — on Windows, set up LibreHardwareMonitor as the fallback. Linux non-NVIDIA support is on the roadmap.
  • WSL / VM? NVML isn't available inside WSL2 without --runtime=nvidia. Run dofek on the Windows host instead.
  • Apple Silicon (M-series)? Expected. The GPU tile shows your chip name (e.g. "Apple M4 GPU") with a unified-memory hint pointing to the MEM panel — Apple Silicon GPU/VRAM and CPU temperature integration are tracked for a future release. Apple's GPU shares system memory, so VRAM usage shows up as memory pressure rather than a separate number.

Ollama plugin not showing models

  • Verify Ollama is running: curl http://localhost:11434/api/tags
  • Check the plugin dock — is it green (●) or red (●)?
  • If red, the plugin likely crashed. Run the plugin binary directly to see its stderr output: dofek-ollama.exe from C:\Program Files\dofek\ on Windows, dofek-ollama from /usr/bin/ or the AppImage on Linux, or dofek-ollama from inside dofek.app/Contents/MacOS/ on macOS.
  • Confirm the plugin is enabled in dofek.toml under [[plugins]].

TUI looks cramped or wraps awkwardly

  • Set terminal font size to 9–11pt (JetBrains Mono, Fira Code, MesloLGS NF, Cascadia Code, or Consolas all work).
  • Minimum terminal width ~110 columns, height ~30 rows. Below that, dofek shows a size hint.
  • In Windows Terminal, run install-wt-profile.ps1 (from the source repo) for a pre-configured profile. On Linux/macOS, set the font in your terminal preferences.

CPU temperature always shows as "—"

  • Windows: CPU temperature sensors aren't exposed to user-space processes without elevation. Enable LibreHardwareMonitor (see above) to get temperatures via its HTTP fallback.
  • Linux: dofek reads /sys/class/hwmon via sysinfo::Components. If your kernel doesn't expose package-level temps, only per-core sensors will appear; ensure the relevant coretemp/k10temp/amdgpu module is loaded.
  • macOS: CPU temperature/power and GPU/VRAM are not implemented in v1.3 — Apple Silicon SMC integration is tracked for a future release.

Process kill fails with permission errors

  • Windows ("Access is denied"): system services, antivirus components, and protected processes can only be killed as admin. Launch dofek from an elevated prompt if needed; running it elevated isn't generally recommended.
  • Linux/macOS ("Operation not permitted"): SIGTERM is sent to the target PID. Root-owned processes need sudo dofek-tui. SIP-protected macOS system processes can't be killed at all.
