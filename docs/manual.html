1. What is dofek?

dofek (Hebrew: דּוֹפֶק — "pulse" or "heartbeat") is a Windows-native system monitor built for developers running local AI workloads. It tracks per-process VRAM, flags Ollama and other inference workloads automatically, and renders CPU history as candlestick charts so you see real variance — not smoothed averages.

It ships as two interfaces driven by the same Rust core:

  • dofek-tui — terminal UI (Ratatui). Lives happily inside Windows Terminal, Tabby, or any VT-capable console.
  • dofek-gui — desktop app (Tauri + WebView2). Same data, same plugins, graphical charts.

2. Quickstart

GUI (desktop app)

  1. Launch dofek from the Start Menu, or run dofek.exe from C:\Program Files\dofek\.
  2. The dashboard opens with live CPU, GPU, memory, and network charts. The process watchlist on the right shows the top VRAM/CPU/MEM consumers.
  3. Click AI / DEV / WATCH tabs to filter the watchlist.
  4. Press ? for the keybinding overlay, or click the Manual link inside the help overlay to reopen this document.

TUI (terminal)

  1. Open Windows Terminal (or PowerShell), then run: 
    dofek-tui
  2. With a custom config: 
    dofek-tui --config "%APPDATA%\dofek\dofek.toml"
  3. Press ? for help, q to quit. See the full keybinding table.
Font size tip: the TUI is designed for 9–10pt monospace. JetBrains Mono works well. Run install-wt-profile.ps1 (ships with the source) to add a pre-configured "dofek" profile to Windows Terminal.

3. TUI vs. GUI

Both interfaces read the same config, run the same plugins, and collect the same data. Pick based on context:

Use caseRecommended
Remote session / SSH / low-resourceTUI
Always-visible dashboard on your desktopGUI
Scripting + snapshot exports (s key)TUI
Smoothed Bezier charts, click-to-killGUI
Running alongside ollama, lm-studioEither — both detect AI workloads

5. Configuration

dofek loads config from the first file it finds, in this order:

  1. --config <path> command-line flag (TUI only)
  2. ./dofek.toml in the current working directory
  3. %APPDATA%\dofek\dofek.toml — recommended location

All sections are optional. Missing values fall back to sensible defaults.

Example config

[general]
refresh_ms = 500          # Poll interval (milliseconds)
history_len = 60          # Chart samples to keep in memory

[display]
show_temps = true
show_power = true
process_count = 50        # Max processes in watchlist

[ai]
vram_threshold_gb = 1.0   # VRAM above this flags a process as AI
known_ai_processes = ["ollama", "ollama_llama_server", "python", "lm_studio", "claude"]

[categories]
dev_processes   = ["code", "cargo", "rustc", "node", "npm", "git", "docker", "go"]
watch_processes = ["ffmpeg", "obs"]
watch_pids      = []

[lhm]
url = "http://localhost:8085"   # LibreHardwareMonitor web server (fallback)

[telemetry]
enabled = false           # Opt-in; disabled by default

[[plugins]]
name       = "ollama"
command    = "dofek-ollama"
args       = ["--host", "http://localhost:11434"]
enabled    = true
timeout_ms = 2000

Option reference

KeyDescription
general.refresh_msPoll interval. Lower = smoother, more CPU. Default 500.
general.history_lenSamples retained per chart. Default 60.
display.show_tempsShow temperature bars in the bottom strip.

[[plugins]]Repeatable table. Each entry spawns a plugin child process.

6. Process categories (AI / DEV / WATCH)

Every process gets classified into zero or one category. Priority goes WATCH → AI → DEV → none.

CategoryHow it's assigned
AI ●Name matches ai.known_ai_processes, or VRAM > ai.vram_threshold_gb, or name ends with _server and uses any VRAM.
DEV ■Name matches categories.dev_processes.
WATCH ★Name matches categories.watch_processes, or PID listed in categories.watch_pids.

Filter the watchlist with keys 1 (ALL) / 2 (AI) / 3 (DEV) / 4 (WATCH), or click the tabs in the GUI.

7. AI workload detection

Once a process is classified as AI, dofek tags its current state:

BadgeCondition
● inferringVRAM > threshold and GPU utilization > 20%.
● loadingVRAM increasing by > 200 MB between polls.
○ idleKnown AI process, low utilization.

The AI badge in the top ticker bar always reflects the currently-most-active AI workload.

8. GPU support & LibreHardwareMonitor

dofek talks to two data sources for GPU metrics:

  1. NVML (primary) — NVIDIA driver API. Provides utilization, VRAM, temperature, power, and per-process VRAM. No extra install required beyond the NVIDIA driver.
  2. LibreHardwareMonitor HTTP (fallback) — for non-NVIDIA GPUs or when NVML isn't available.

Setting up LibreHardwareMonitor

  1. Download LibreHardwareMonitor and extract.
  2. Run LibreHardwareMonitor.exe as administrator.
  3. Enable the web server: Options → Remote Web Server → Run (default port 8085).
  4. If you changed the port, update lhm.url in dofek.toml.
Multi-GPU: NVML is fully multi-GPU aware. LHM support is basic — primary GPU only. If you have multiple GPUs and NVML isn't working, open an issue.

9. Plugins

Plugin API is experimental. The JSON protocol is versioned (schema_version: 1) but subject to breaking changes until further notice. Pin your plugin to a specific dofek version if stability matters.

Plugins are external processes that dofek spawns and polls via newline-delimited JSON over stdin/stdout. Each poll cycle, dofek writes a request, reads a response, and renders any panels / process annotations / metrics the plugin returned.

Built-in plugins

PluginPurposeRequires
dofek-ollamaShows loaded models + inference state, annotates Ollama processesOllama running locally
dofek-dockerLists running containers, annotates Docker processesDocker Desktop with TCP API enabled

Enable a plugin by adding a [[plugins]] table to dofek.toml. See the example config above.

Plugin health dock

The plugin dock (inside the watchlist panel) shows one row per plugin:

  • ● healthy
  • ● unhealthy (5+ consecutive errors)
  • ● crashed — dofek restarts with exponential backoff (1s → 30s cap)
  • ○ starting

Plugin development docs: dofek.dev/plugins.

10. Telemetry

dofek includes opt-in anonymous telemetry to help improve the app. It is disabled by default. Nothing is sent unless you explicitly enable it.

What's collected (if you opt in)

  • Session duration and interface (TUI vs. GUI)
  • Feature usage: which tabs, filters, modes you use
  • GPU detection path (NVML / LHM / none) and device count
  • Periodic heartbeats (process count, active tab)

What's never collected

  • Process names, hostnames, IP addresses, or any identifying information
  • System metric values (CPU %, memory usage, etc.)
  • Config file contents or custom process lists

A random anonymous UUID is generated on first run and stored at %APPDATA%\dofek\settings.toml. Events batch locally and flush via HTTPS every 60 seconds. If the endpoint is unreachable, batches are silently dropped.

To disable: toggle the switch in the GUI help overlay, or set telemetry.enabled = false in dofek.toml.

11. Troubleshooting

SmartScreen warning on install or first run

The v1.0 binaries are unsigned. Code signing is on the post-1.0 roadmap. Workaround:

  1. Right-click the installer or executable → Properties.
  2. Check Unblock at the bottom of the General tab.
  3. Click OK, then run as normal.

Verify the download first by comparing against SHA256SUMS.txt from the GitHub release:

Get-FileHash .\dofek_1.0.0_x64_en-US.msi -Algorithm SHA256

GPU panel shows "no devices"

  • NVIDIA GPU? Make sure drivers are current. nvidia-smi should work from a terminal.
  • AMD / Intel GPU? NVML won't work — set up LibreHardwareMonitor as the fallback.
  • WSL / VM? NVML isn't available inside WSL2 without --runtime=nvidia. Run dofek on the Windows host instead.

Ollama plugin not showing models

  • Verify Ollama is running: curl http://localhost:11434/api/tags
  • Check the plugin dock — is it green (●) or red (●)?
  • If red, the plugin likely crashed. Run dofek-ollama.exe directly from C:\Program Files\dofek\ to see its stderr output.
  • Confirm the plugin is enabled in dofek.toml under [[plugins]].

TUI looks cramped or wraps awkwardly

  • Set terminal font size to 9–10pt (JetBrains Mono, Cascadia Code, or Consolas all work).
  • Minimum terminal width ~110 columns, height ~30 rows. Below that, dofek shows a size hint.
  • In Windows Terminal, run install-wt-profile.ps1 (from the source repo) for a pre-configured profile.

CPU temperature always shows as "—"

On Windows, CPU temperature sensors aren't exposed to user-space processes without elevation. Enable LibreHardwareMonitor (see above) to get temperatures via its HTTP fallback.

Process kill fails with "Access is denied"

Some processes (system services, antivirus components, protected processes) can only be killed as admin. Launch dofek from an elevated prompt if you need that capability. Running dofek elevated is not generally recommended.
