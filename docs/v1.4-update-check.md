# v1.4 — Check for Updates (notify-only)

Adds a manual + opt-in startup "is there a newer Dofek?" check. Notify-only — no
download, no installer, no signing infrastructure required. Bigger auto-update
work (Tauri updater plugin, code-signing on macOS, etc.) is deferred.

## Source of truth

GitHub Releases API:
`https://api.github.com/repos/AsafSaar/dofek/releases/latest`

Compare `tag_name` (e.g. `v1.3.4`) against the compiled-in `CARGO_PKG_VERSION`.
Unauthenticated requests get 60/hr/IP — plenty for human-triggered checks.

## Core module — `src/update.rs`

```rust
pub struct UpdateInfo {
    pub current: String,   // CARGO_PKG_VERSION
    pub latest: String,    // tag_name minus leading "v"
    pub is_newer: bool,
    pub url: String,       // html_url of release
    pub notes: String,     // body, truncated to ~600 chars
}
pub fn check() -> anyhow::Result<UpdateInfo>;
```

- Uses existing `ureq` (already in Cargo.toml with native-tls).
- Sets `User-Agent: dofek/<version>` (GitHub rejects unauth requests w/o UA).
- 5 s timeout. Failures bubble up as `anyhow::Error` — caller decides whether to
  surface or swallow.
- Semver compare done inline (parse `MAJOR.MINOR.PATCH`, ignore pre-release
  suffixes). No new crate dependency.

## Surfaces

### TUI
- New key `u` → kicks off an async check on a worker thread; overlay shows
  `Checking…` → result.
- Overlay (`src/ui/update.rs`) shows current vs latest, release URL, first few
  lines of release notes, plus "press any key to close".
- Result cached on `App` until the user closes the overlay; subsequent `u`
  re-runs the check.
- Help screen gets a `u` row.

### GUI
- New Tauri command `check_for_update() -> Result<UpdateInfo, String>`.
- "Check for updates" button in the settings overlay.
- If a newer version is found, the topbar version pill gets a green dot + click
  opens the release URL via `open_manual`-style shell open.
- On startup (if `check_updates_on_startup`), runs the check off the main
  thread; if newer, shows a toast with a clickable link.

### Settings
Add to `UserSettings`:
- `check_updates_on_startup: bool` (default `false` — opt-in).

Exposed in:
- GUI settings overlay (toggle).
- `dofek.toml.example` (documented).
- TUI: read-only via settings file for now (no in-TUI toggle UI yet).

## Out of scope
- Auto-download / auto-install. Requires signing on macOS + Tauri updater
  plugin + a hosted update manifest. Tracked separately.
- Pre-release / beta channel selection.
- Throttling / dedup of repeated startup checks (rate limit is generous;
  revisit if telemetry shows it matters).
